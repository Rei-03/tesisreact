## 📝 CONSULTAS ENCONTRADAS:
**Consulta de circuitos apagables:**  
```sql
SELECT idCircuitoP, idProv, Circuito33, Bloque, CircuitoP, Clientes, ZonaAfectada, Apagable
FROM ap_circuitos
WHERE Apagable = 1
ORDER BY CircuitoP
```  
Propósito: Mostrar circuitos disponibles para selección en formularios (ej: Aseguramientos)  
Tablas: `ap_circuitos`  
Filtros: `Apagable = 1`  

**Consulta de aseguramientos por fecha:**  
```sql
SELECT id_CircuitoP, CircuitoP, fechaInicial, fechaFinal, Observaciones, mw, tipo
FROM ap_Aseguramientos
WHERE fechaInicial <= @fecha AND fechaFinal >= @fecha
ORDER BY fechaFinal DESC
```  
Propósito: Mostrar aseguramientos activos en pantalla principal  
Tablas: `ap_Aseguramientos`  
Filtros: `fechaInicial <= @fecha AND fechaFinal >= @fecha`  

**Consulta de próximas aperturas:**  
```sql
SELECT id_Circuito, Circiuto, [MW Hora Actual], Bloque FROM ap_ProxAperturas
```  
Propósito: Mostrar próximas aperturas en pantalla principal  
Tablas: `ap_ProxAperturas`  
Filtros: Ninguno  

**Procedimiento almacenado ap_lisApagonCFG:**  
Propósito: Listar afectaciones por circuito y fecha  
Parámetros: `@idProv varchar(3)`, `@Fecha smalldatetime`  
Retorna: `idCircuitoP`, `Bloque`, `Circuito33`, `CircuitoP`, `Clientes`, `MWAfectados`, `FechaRetiro`, `FechaCierre`, `Tiempo`, `ZonaAfectada`, `Observaciones`  

**Procedimiento almacenado ap_mwxHora:**  
Propósito: Obtener MW por circuito y fecha  
Parámetros: `@idCircuitoP int`, `@Fecha datetime`  
Retorna: `mw decimal`  

## 🌐 ENDPOINTS/ACCESOS:
**Acceso a datos de circuito:**  
Método: `ap_circuitosTableAdapter.Fill()` en `Aseguramientos.cs` y `Form1.cs`  
HTTP: N/A (WinForms, no APIs)  
Lógica: Carga DataSet desde DB, filtra `Apagable = 1`, ordena por `CircuitoP`  

**Acceso a afectaciones de circuito:**  
Método: `ap_lisApagonCFGTableAdapter.Fill()` en `Form1.cs`  
HTTP: N/A  
Lógica: Ejecuta SP `ap_lisApagonCFG` con `idProv="CFG"` y fecha actual  

**Acceso a aseguramientos:**  
Método: `ap_AseguramientosTableAdapter.Fill()` en `Form1.cs` y `Aseguramientos.cs`  
HTTP: N/A  
Lógica: Carga aseguramientos por fecha, calcula MW total afectado  

**Acceso a próximas aperturas:**  
Método: `ap_ProxAperturasTableAdapter.Fill()` en `Form1.cs`  
HTTP: N/A  
Lógica: Carga todas las aperturas, agrupa por bloque para calcular MW
